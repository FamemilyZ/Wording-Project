# Technology Stack Justification (เหตุผลในการเลือกใช้เทคโนโลยี)

เอกสารนี้อธิบายถึงเทคโนโลยีที่เลือกใช้สำหรับโครงการ **Wording** ซึ่งเป็นเว็บแอปพลิเคชันแบบ Client-side ทั้งหมด (ทำงานบนเบราว์เซอร์ของผู้ใช้)

## ภาพรวมสถาปัตยกรรม

แอปพลิเคชันนี้เป็นแบบ **Static Web Application** ที่ไม่จำเป็นต้องมี Back-end Server เป็นของตัวเองในการทำงานหลัก แต่มีการเรียกใช้ External API เพื่อเสริมฟังก์ชันการทำงาน

- **Frontend (Client-side):** เป็นส่วนหลักของระบบ ประกอบด้วย HTML, CSS, และ JavaScript ทำหน้าที่แสดงผลหน้าจอทั้งหมด, จัดการ Logic ของเกม, และประมวลผลการทำงานส่วนใหญ่
- **External Services:**
    - **Translation API:** เรียกใช้บริการแปลภาษาจากภายนอก (`mymemory.translated.net`) เพื่อสร้างคลังคำศัพท์แบบกำหนดเอง
    - **Image Hosting:** รูปภาพสำหรับคำศัพท์ถูกจัดเก็บและเรียกใช้จาก Google Cloud Storage ซึ่งเป็นบริการภายนอก
- **Storage:** ใช้ **Browser Local Storage** ในการจัดเก็บข้อมูลประวัติการเล่นของผู้ใช้ ซึ่งเป็นวิธีที่ง่ายและเหมาะสมสำหรับข้อมูลที่ไม่ซับซ้อนและไม่ต้องการการซิงค์ข้ามอุปกรณ์

## เทคโนโลยีที่เลือกใช้ (Selected Stack)

### 1. Frontend

- **ภาษาโปรแกรม: JavaScript (ES6+)**
    - **เหตุผล:**
        - **ความเข้ากันได้สากล (Universal Compatibility):** JavaScript เป็นภาษามาตรฐานสำหรับเว็บเบราว์เซอร์ทุกชนิด ทำให้มั่นใจได้ว่าแอปพลิเคชันจะทำงานได้สำหรับผู้ใช้ทุกคนโดยไม่ต้องติดตั้งส่วนเสริมใด ๆ
        - **ไม่ต้องพึ่งพา Framework (Vanilla JS):** สำหรับโครงการที่มีขอบเขตชัดเจนและไม่ซับซ้อนมากนัก การใช้ Vanilla JavaScript (JS บริสุทธิ์) ช่วยให้แอปพลิเคชันมีขนาดเล็ก โหลดเร็ว และง่ายต่อการทำความเข้าใจโครงสร้างโค้ดทั้งหมดโดยไม่ต้องเรียนรู้ Framework เพิ่มเติม
        - **DOM Manipulation:** JavaScript มีความสามารถในการจัดการ Document Object Model (DOM) ได้อย่างดีเยี่ยม ทำให้การสร้างและอัปเดตหน้าจอแบบไดนามิก (เช่น การแสดงคำถาม, ตัวเลือก, ผลลัพธ์) เป็นไปได้อย่างมีประสิทธิภาพ
        - **Asynchronous Operations:** สามารถเรียกใช้ API จากภายนอก (เช่น API การแปล) ได้อย่างง่ายดายด้วย `fetch` โดยไม่ขัดขวางการทำงานหลักของโปรแกรม

- **โครงสร้าง: HTML5**
    - **เหตุผล:**
        - **Semantic HTML:** ใช้ Tag ที่มีความหมาย (เช่น `<header>`, `<main>`, `<div>` ที่มี id ชัดเจน) ทำให้โครงสร้างของเอกสารเข้าใจง่าย และเป็นประโยชน์ต่อ SEO และ Accessibility
        - **มาตรฐานเว็บ:** เป็นเวอร์ชันล่าสุดของ HTML ที่รองรับโดยเบราว์เซอร์สมัยใหม่ทั้งหมด

- **การออกแบบ: CSS3**
    - **เหตุผล:**
        - **Rich Styling Features:** มาพร้อมกับคุณสมบัติใหม่ ๆ เช่น Flexbox, Grid, Custom Properties (Variables), และ Transitions/Animations ซึ่งช่วยให้สามารถสร้าง Layout ที่ซับซ้อนและ User Interface ที่สวยงาม ตอบสนองได้ดี (Responsive)
        - **การแยกส่วนของโค้ด (Separation of Concerns):** การแยกโค้ด CSS ออกจาก HTML และ JavaScript ช่วยให้การดูแลรักษาโค้ดในระยะยาวทำได้ง่ายขึ้น

### 2. External API

- **บริการ: MyMemory Translation API (`api.mymemory.translated.net`)**
    - **เหตุผล:**
        - **ใช้งานฟรีและไม่ต้องลงทะเบียน:** เป็น API ที่สามารถเรียกใช้งานได้ทันทีโดยไม่ต้องใช้ API Key ทำให้ง่ายต่อการทดลองและพัฒนาสำหรับโครงการขนาดเล็ก
        - **ตอบโจทย์ฟังก์ชัน "Custom Mode":** ตอบสนองความต้องการของ User Story ที่ต้องการให้ผู้ใช้สร้างควิซของตัวเองจากคำศัพท์ที่ป้อนเข้ามาได้อย่างสมบูรณ์

### 3. Storage (พื้นที่จัดเก็บข้อมูล)

- **เทคโนโลยี: Browser Local Storage**
    - **เหตุผล:**
        - **ความเรียบง่าย:** เป็น API ที่มีอยู่ในเบราว์เซอร์ทุกตัว ใช้งานง่ายเพียงไม่กี่บรรทัดในการบันทึกและอ่านข้อมูล เหมาะสมอย่างยิ่งสำหรับฟีเจอร์ "ดูประวัติการเล่น" (View History) ซึ่งเป็นข้อมูลฝั่งผู้ใช้เท่านั้น
        - **การทำงานแบบ Offline:** ข้อมูลจะถูกเก็บไว้ในเครื่องของผู้ใช้ ทำให้สามารถเรียกดูประวัติได้แม้ไม่มีการเชื่อมต่ออินเทอร์เน็ต
        - **เพียงพอต่อความต้องการ:** สำหรับโครงการนี้ที่ต้องการเก็บข้อมูลขนาดเล็ก (ข้อความ) และไม่ต้องการความปลอดภัยระดับสูงหรือการซิงค์ข้อมูลระหว่างอุปกรณ์ Local Storage จึงเป็นตัวเลือกที่เหมาะสมที่สุด
