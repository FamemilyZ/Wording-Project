# เหตุผลในการเลือกใช้เทคโนโลยี (Technology Stack Justification)

เอกสารนี้อธิบายเหตุผลเบื้องหลังการเลือกใช้เทคโนโลยีสำหรับโครงการ Vocabulary Quiz Generator

## 1. Front-end (Client-Side)

### **HTML5**
- **เหตุผล**: เป็นภาษามาร์กอัปมาตรฐานสำหรับเว็บแอปพลิเคชัน มีองค์ประกอบเชิงความหมาย (Semantic Elements) ที่ช่วยในเรื่อง SEO และ Accessibility

### **CSS3**
- **เหตุผล**: ใช้สำหรับจัดรูปแบบและดีไซน์หน้าตาของเว็บให้สวยงาม คุณสมบัติใหม่ๆ เช่น Flexbox, Grid, และ Custom Properties (Variables) ทำให้การสร้าง Responsive Layout และการจัดการธีมสีทำได้ง่ายและมีประสิทธิภาพ

### **Vanilla JavaScript (ES6+)**
- **เหตุผล**:
    - **No Dependencies**: การไม่ใช้เฟรมเวิร์กหรือไลบรารีภายนอกทำให้แอปพลิเคชันมีขนาดเล็ก โหลดเร็ว และไม่ต้องกังวลเรื่องการจัดการ Dependencies
    - **Performance**: โค้ดที่เขียนด้วย Vanilla JS มักจะทำงานได้เร็วกว่าเมื่อเทียบกับการใช้เฟรมเวิร์กที่มี Overhead
    - **Sufficient for Scope**: สำหรับขอบเขตของโปรเจกต์นี้ ซึ่งเน้นการจัดการ DOM, จัดการ Events, และโต้ตอบกับ Local Storage นั้น Vanilla JS มีความสามารถเพียงพอโดยไม่จำเป็นต้องใช้เครื่องมือที่ซับซ้อนเกินความจำเป็น

## 2. การจัดเก็บข้อมูล (Data Storage)

### **Browser Local Storage**
- **เหตุผล**: เลือกใช้ Local Storage สำหรับการบันทึกประวัติการเล่นและด่านที่ผู้ใช้สร้างเอง เนื่องจากเป็นวิธีที่ง่ายและไม่ต้องพึ่งพา Server ภายนอก ข้อมูลจะถูกเก็บไว้ในเบราว์เซอร์ของผู้ใช้โดยตรง เหมาะสำหรับแอปพลิเคชันขนาดเล็กที่ไม่ต้องการระบบบัญชีผู้ใช้หรือการซิงค์ข้อมูลข้ามอุปกรณ์
