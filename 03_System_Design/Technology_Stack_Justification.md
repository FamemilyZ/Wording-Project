# เหตุผลในการเลือกใช้เทคโนโลยี (Technology Stack Justification)

## 1. Frontend (ส่วนหน้าบ้าน)

- **เทคโนโลยี:** HTML, CSS, และ JavaScript (Vanilla JS)
- **เหตุผล:**
    - **ความเรียบง่ายและพื้นฐาน:** สำหรับโปรเจกต์ขนาดเล็กที่ไม่ซับซ้อน การใช้ Vanilla JS (JavaScript เพียวๆ) โดยไม่มี Framework ทำให้เริ่มต้นได้รวดเร็ว ลดความจำเป็นในการเรียนรู้เครื่องมือใหม่ และทำให้ขนาดของโปรเจกต์โดยรวมเล็ก
    - **ประสิทธิภาพ:** การไม่ใช้ Framework ทำให้ไม่มี Overhead ที่ไม่จำเป็น ส่งผลให้หน้าเว็บโหลดได้เร็วขึ้น ซึ่งสำคัญต่อประสบการณ์ผู้ใช้
    - **เข้ากันได้สากล:** เป็นเทคโนโลยีมาตรฐานของเว็บที่สามารถทำงานได้บนทุกเบราว์เซอร์สมัยใหม่โดยไม่ต้องมีการตั้งค่าที่ซับซ้อน

## 2. Backend & Infrastructure (ส่วนหลังบ้านและโครงสร้างพื้นฐาน)

- **เทคโนโลยี:** Firebase (Platform as a Service - PaaS)
- **เหตุผล:**
    - **Firebase Authentication:** เป็นบริการยืนยันตัวตนที่ปลอดภัยและติดตั้งง่าย รองรับผู้ให้บริการหลายเจ้า (Email/Password, Google) ช่วยลดภาระในการพัฒนาระบบสมาชิกที่ซับซ้อนและดูแลเรื่องความปลอดภัย
    - **Cloud Firestore:** เป็นฐานข้อมูลแบบ NoSQL ที่มีความยืดหยุ่นสูง เหมาะสำหรับการเก็บข้อมูลผู้ใช้, คะแนน, ประวัติการเล่น, และชุดคำศัพท์ที่ผู้ใช้สร้างเองซึ่งอาจมีโครงสร้างไม่ตายตัว การเชื่อมต่อกับ Frontend ทำได้ง่ายผ่าน Firebase SDK
    - **Firebase Hosting:** เป็นบริการโฮสติ้งสำหรับ Static Web ที่รวดเร็วและปลอดภัย มี CDN (Content Delivery Network) ในตัว ทำให้ผู้ใช้ทั่วโลกเข้าถึงเว็บได้เร็ว พร้อมทั้งมี Free Tier ที่เพียงพอสำหรับโปรเจกต์ขนาดนักศึกษา และขั้นตอนการ Deploy ที่ง่ายมากเพียงใช้คำสั่งเดียว

## สรุป
การผสมผสานระหว่าง Vanilla JS และ Firebase เป็นทางเลือกที่เหมาะสมสำหรับโปรเจกต์ "Wording" เนื่องจากเน้นที่การพัฒนาที่รวดเร็ว (Rapid Development) ลดความซับซ้อนของโครงสร้าง และใช้ประโยชน์จากบริการสำเร็จรูป (Managed Services) เพื่อให้สามารถมุ่งเน้นไปที่การพัฒนาฟังก์ชันหลักของแอปพลิเคชันได้เต็มที่
