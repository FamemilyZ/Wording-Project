# System Architecture Diagram (แผนภาพสถาปัตยกรรม)

ไฟล์นี้ควรเป็นไฟล์รูปภาพ (`.png`) ที่แสดงสถาปัตยกรรมของระบบในภาพรวม

## สิ่งที่ควรมีในแผนภาพ (C4 Model - Level 1 or 2)

### องค์ประกอบหลัก (Components)
1.  **User (ผู้ใช้):** ผู้ที่ใช้งานแอปพลิเคชันผ่านเว็บเบราว์เซอร์
2.  **Web Browser (Client):** Single-Page Application (SPA) ที่เขียนด้วย HTML, CSS, JS ทำหน้าที่แสดงผลและรับ Input จากผู้ใช้
3.  **Firebase (Backend Platform):** แพลตฟอร์มที่ให้บริการหลังบ้านทั้งหมด ประกอบด้วย:
    - **Firebase Hosting:** ให้บริการไฟล์ของเว็บแอปพลิเคชัน (HTML, CSS, JS) และทำหน้าที่เป็น CDN
    - **Firebase Authentication:** จัดการการลงทะเบียนและเข้าสู่ระบบของผู้ใช้
    - **Cloud Firestore:** ฐานข้อมูลสำหรับเก็บข้อมูลผู้ใช้, คะแนน, และด่านที่สร้างเอง

### การเชื่อมต่อ (Connections)
- **User -> Web Browser:** ผู้ใช้โต้ตอบกับเว็บแอปพลิเคชัน
- **Web Browser -> Firebase Hosting:** เบราว์เซอร์ดาวน์โหลดไฟล์หน้าเว็บจาก Hosting
- **Web Browser -> Firebase Authentication:** เบราว์เซอร์ส่งข้อมูลการล็อกอิน/ลงทะเบียนเพื่อยืนยันตัวตน
- **Web Browser -> Cloud Firestore:** เบราว์เซอร์ทำการอ่านและเขียนข้อมูล (เช่น บันทึกคะแนน, โหลดด่าน) โดยตรงผ่าน Firebase SDK

**หมายเหตุ:** Gemini ไม่สามารถสร้างไฟล์รูปภาพได้ กรุณาสร้างแผนภาพนี้ด้วยเครื่องมือเช่น Draw.io, Figma, หรือโปรแกรมอื่นๆ แล้วบันทึกเป็นไฟล์ `Architecture_Diagram.png` มาไว้ในโฟลเดอร์นี้
